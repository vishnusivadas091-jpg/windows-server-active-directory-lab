01 — Project Overview
What This Project Covers

This lab simulates a real-world enterprise IT environment built from the ground up. It demonstrates hands-on proficiency across six key infrastructure domains: server provisioning, identity management, network services, policy enforcement, file access control, and client management. Every step was documented with screenshots to show the complete operational process — directly relevant to ITAM, helpdesk, and systems administration roles.

🖥️
Phase 1
Server Setup
Installed Windows Server 2022 Standard (Desktop Experience), configured the admin account, renamed the server to DC01, and launched Server Manager.
🏗️
Phase 2
AD DS & DNS
Added Active Directory Domain Services and DNS roles, promoted DC01 to domain controller, and created the lab.local forest with a new root domain.
👥
Phase 3
Users & OUs
Built Organizational Units (IT, HR, Sales), created domain users in each OU, created security groups, and assigned users to their respective groups.
🌐
Phase 4
DHCP Server
Installed the DHCP Server role, configured a scope for 192.168.10.0/24, set exclusions, assigned the default gateway, and activated the scope.
🔒
Phase 5
Group Policy
Created the "Lab security IT policy" GPO, configured password policies, account lockout rules, and screen saver enforcement. Validated with gpupdate /force.
📁
Phase 6
File Server & Client
Deployed File Server Resource Manager, created department shares with NTFS and share permissions, joined a Windows 11 client, and verified all access end-to-end.
02 — Implementation Walkthrough
Complete Step-by-Step Timeline
01
Phase 1 — OS Installation
Windows Server 2022 Installation
Selected Windows Server 2022 Standard (Desktop Experience) on a 60.2 GB virtual disk via the Microsoft Server Operating System Setup wizard. Configured the built-in Administrator password and confirmed first boot into Server Manager with initial roles visible.
Edition: Standard (Desktop Experience)Disk: 60.2 GBArch: x64
02
Phase 1 — Server Config
Server Rename to DC01
Renamed the server from the default random hostname (WIN-KS2TRSUOD95) to DC01 via Settings → System → Rename this PC. This naming convention clearly identifies the machine's intended role as Primary Domain Controller before AD DS is installed.
Hostname: DC01Role: Primary Domain Controller
03
Phase 2 — AD DS Role
Installing Active Directory Domain Services
Used the Add Roles and Features Wizard to install Active Directory Domain Services, Group Policy Management, Remote Server Administration Tools, AD DS Snap-Ins, and the Active Directory module for Windows PowerShell — all on DC01.
Role: AD DSGroup Policy MgmtRSAT + AD Module
04
Phase 2 — Domain Promotion
Promoting DC01 to Domain Controller (lab.local)
Launched the AD DS Configuration Wizard and selected "Add a new forest". Set root domain name to lab.local. Configured domain controller options: Windows Server 2016 functional level, DNS Server enabled, Global Catalog enabled, DSRM password set. All prerequisite checks passed — server rebooted and promoted successfully.
Domain: lab.localForest Level: WS 2016DNS: EnabledGC: Enabled
05
Phase 3 — AD Structure
OUs, Users & Security Groups
Opened ADUC and verified the lab.local domain. Created three Organizational Units — IT, HR, and Sales — under the domain root. Populated OUs with users: John Smith (IT), Mike Brown (Sales), Emma Wilson (Sales). Created the Sales-Team security group and added Mike Brown as a member, confirming via the Members tab.
OUs: IT, HR, SalesUsers: John Smith, Mike Brown, Emma WilsonGroup: Sales-Team
06
Phase 4 — DHCP
DHCP Server Role & Scope Configuration
Installed DHCP Server on DC01.lab.local. Created a new IPv4 scope covering 192.168.10.1–192.168.10.100 (/24 subnet). Added an exclusion for .1–.10 (reserved for static infrastructure). Configured scope options: assigned default gateway 192.168.10.254. Activated the scope. Verified the DHCP console showing Address Pool, Leases, Reservations, Scope Options, and Policies nodes.
Range: 192.168.10.1–.100Exclusion: .1–.10Gateway: 192.168.10.254Subnet: /24
07
Phase 5 — Group Policy
GPO — Password & Lockout Policies
Created GPO "Lab security IT policy" linked to lab.local. Edited under Computer Configuration → Windows Settings → Security Settings → Account Policies. Set Password Policy: 5-password history, 90-day max age, 30-day min age, 8-character min length, complexity enabled. Set Lockout Policy: 30-minute duration, 5 failed attempts threshold, 30-minute reset counter.
History: 5 passwordsMax Age: 90 daysMin Length: 8 chars Complexity: OnLockout: 5 attempts / 30 min
08
Phase 5 — GPO Personalization
Screen Lock Policy & gpupdate /force
Under User Configuration → Admin Templates → Control Panel → Personalization: enabled screen saver, password-protected screen saver, and screen saver timeout. Ran gpupdate /force from Administrator command prompt — confirmed "Computer Policy update has completed successfully. User Policy update has completed successfully."
Screen Saver: EnabledPassword Lock: EnabledApplied: gpupdate /force
09
Phase 6 — File Server
File Server Resource Manager & Department Shares
Installed File Server Resource Manager on DC01. Created three folders on C:\shares\: HR, IT, and Sales. Assigned Share Permissions on Sales: LAB\Sales-Team granted Full Control, Change, and Read. Configured NTFS Security on IT folder: assigned rights to CREATOR OWNER, SYSTEM, IT-Team (LAB\IT-Team), and LAB\Administrators — enforcing least-privilege access.
Shares: C:\shares\HR, IT, SalesShare Perms: Sales-Team Full ControlNTFS: IT-Team scoped
10
Phase 6 — Client VM
Windows 11 Pro Installation & Domain Join
Installed Windows 11 Pro (required for domain join) on a second VM. Joined WIN11-PC to lab.local via System Properties → Computer Name/Domain Changes. Received the "Welcome to the lab.local domain" confirmation. Logged in as John Smith (Jsmith@lab.local) — Windows 11 Start Menu confirmed domain account active.
Client: WIN11-PCDomain: lab.localLogin: Jsmith@lab.local
11
Phase 6 — Verification
Share Access & GPO Verification (gpresult /r)
From WIN11-PC, accessed \\192.168.10.254 via Run dialog — HR, IT, Sales, netlogon, and sysvol shares all visible. Ran gpresult /r as jsmith: confirmed Domain: LAB, Applied GPO: "Lab security IT policy", and IT-Team security group membership — full end-to-end policy and access validation.
Shares visible: HR, IT, SalesGPO: Lab security IT policyGroup: IT-Team confirmed
12
Phase 6 — User Lifecycle
AD User Lifecycle Management Tasks
Demonstrated key helpdesk/ITAM tasks in ADUC: unlocked John Smith's account (Account tab → Unlock account), performed a password reset (Reset Password dialog showing account status: Unlocked), created new user Emma Wilson (Ewilson@lab.local, Sales OU, must change password at next logon), and disabled John Smith's account — confirmed "Object John Smith has been disabled." Verified DHCP lease via ipconfig /release and /renew — client received 192.168.10.11 with gateway 192.168.10.254.
Unlock: John SmithPassword ResetNew User: Emma WilsonDisable: John SmithDHCP Lease: 192.168.10.11
